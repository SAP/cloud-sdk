Entities can only be updated or deleted if [ETags](#handling-of-etags) match.
If you want to force either an update or the deletion of the entity regardless of the ETag, configure the request to ignore version identifiers with the `ignoreVersionIdentifier()` method.

Below is an example with an update:

```ts
const { businessPartnerApi } = businessPartnerService();
businessPartnerApi
  .requestBuilder()
  .update(businessPartner)
  .ignoreVersionIdentifier();
```

You can also overwrite ETags using method `setVersionIdentifier()`:

```ts
const { businessPartnerApi } = businessPartnerService();
businessPartnerApi
  .requestBuilder()
  .update(businessPartner)
  .setVersionIdentifier('etag');
```

<!-- vale off -->

In the example above, the ETag `etag` is used instead of the original one.
