Running Documentation Locallyâ€‹

Install Node.js and npm if necessary. You can use the latest LTS release.

$ node -v
v12.16.1
$ npm -v
6.14.4

Clone the repository.

git clone git@github.com:SAP/cloud-sdk.git
cd cloud-sdk
git checkout documentation

Run documentation locally.

npm install
npm start

You should see the following output in your console and documentation should automatically open in your browser.

> sap-cloud-sdk-documentation@0.1.0 start /home/USERNAME/dev/documentation
> docusaurus start

Create a branch for the thing you document.

Conventionsâ€‹

  • Static content resides in the static folder.
  • Linking to files and images from Markdown starts from the folder where your file is located. In this case, if I want to link anything in the docs folder I'll have to go with my-link. You can reach up until the root docs folder level with this approach.
  • To make a dynamic link to the static folder content you'll need to use a snippet similar to the one below:
import useBaseUrl from '@docusaurus/useBaseUrl';
<img alt="pull-request" src={useBaseUrl('img/docs/pull-request.png')} />;
  • Of course, you can link with your-link/your-path#your-heading, check the right-side menu for the links like sending you to the top.

Lintersâ€‹

We use linters to catch the most frequent writing issues. Please, use them at all times. The best way to do it, is by adding an extension to your favorite editor.

Valeâ€‹

We use a linter for Vale a linter for prose. It's a powerful tool to keep our writing consistent. Please, check the documentation for more details.

In the situation where linting for grammar and writing style doesn't make sense use the following configuration tags to disable linting for that fragment.

<!-- vale off -->

This is some text

more text here...

<!-- vale on -->

<!-- vale Style.Rule = NO -->

This is some text

<!-- vale Style.Rule = YES -->

Prettierâ€‹

Our Prettier configuration supports Markdown and will help you keep the style consistent. Please, run npm run prettier before you commit your code.
