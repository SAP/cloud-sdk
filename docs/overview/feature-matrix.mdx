## Pregenerated Type-Safe Client Libraries for SAP S/4HANA

:::tip Different packaging strategy

- The SAP Cloud SDK for Java has separate maven modules for SAP S/4HANA Cloud /4HANA On-premise.
  Those are in turn split into modules containing OData v2 and OData v4 services.
  We also ship an aggregating module if you want to add all the SAP S/4HANA client libraries to your dependencies.
  A Java `Class` represents one OData service in SAP S/4HANA.
- The SAP Cloud SDK for JavaScript/TypeScript ships client libraries as separate npm packages for each SAP S/4HANA service.
  Packages for SAP S/4HANA Cloud always contain the latest APIs.
  Packages for SAP S/4HANA On-Premise have `-op-` in their name and contain all the versions of respective service's APIs where the latest version exported as default.
  A `Class` represents an `Entity` in `OData` terms, while each package represents a service.
  The SAP Cloud SDK for JS/TS doesn't ship any aggregator packages.

:::
